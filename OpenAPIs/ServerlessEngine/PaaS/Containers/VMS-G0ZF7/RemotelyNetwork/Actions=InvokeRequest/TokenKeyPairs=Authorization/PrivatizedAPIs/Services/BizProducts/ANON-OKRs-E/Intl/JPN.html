匿名-目標実行管理アプリ 大幅な変更を支援する方法をご確認ください.

未来は、高い目標の継続的な探求にかかっています. 私たちは、達成された目標と主要な結果を通じて、社会のために長期的な価値を構築し、創造することができます. 私たちは匿名であり、私たちに期待しています.

OKR とは?

  • 1. OKR = 目標と主要な結果、目標と主要な結果を意味します;
  • 2. O = 目的は、定性的な幅広い目標の説明であり、企業が期待する方向に進むように促進することです;
    • 2.1 例: 「私たちは何をしたいですか?」、匿名の組織のグローバル ビジネスを成長させるために;
  • 3. KR = Key Result は、決定された目標の達成を測定する主要な結果です;
    • 3.1 例: 「主要な結果が達成されたかどうかをどのように知ることができますか?」、匿名 の 1 億ドルのグローバル売上目標の達成;
  • 4. OKR の概念は、ドラッカーの目標管理に由来します. やりがいのある目標を設定し、コア ワークに焦点を当て、継続的に進捗状況を追跡し、タイムリーにフィードバックを伝達することにより、個人、組織、および会社の目標が最終的に達成されます;

匿名対象実行管理アプリケーションフレームワークを構築します

  • 1. 効果的な (ANON OKR-E) ターゲットの主要な結果を作成し、実行する;
  • 2. (KR) 主要な結果スコアリング システムを確立します;
  • 3. 進捗状況の追跡/継続的なフィードバック/タイムリーな調整;
  • 4. 定期的な業績評価;

匿名ターゲット実行管理アプリケーションを迅速にデプロイ

1. チーム OKR の策定

チーム メンバーは通常、一元化された会議でターゲット クラウドファンディング方式を使用して、チーム OKR を議論および策定し、チームの将来の方向性を明確にします。プロセス全体は 4 つのステップに分かれています。

1.1. チームリーダーは、まずチームの過去の仕事の成果と欠点を体系的に要約し、仕事の方向性を提案します. 紹介の後、すべてのチームメンバーが理解できるように、質問がある場合は全員に尋ねることができます;

1.2. 各チーム メンバーに、チームの目標、個人の仕事の責任、仕事上の関心、およびチームをより良くするためにチームが達成する必要がある目標について考えさせます. 各チーム メンバーは、1~2 個のチーム OKR に貢献します. このプロセス中、チームのメンバーとして、チーム リーダーも 1 ～ 2 個のチーム OKR を提出する必要があります。

1.3. チーム リーダーは、すべてのチーム メンバーから出力されたチーム OKR の提案を要約し、提出者に提出された OKR の価値と意義を 1 つずつ説明するよう求め、他のチーム メンバーの疑問に答えます。すべてのチーム OKR が説明された後、すべてのチーム メンバーが投票し、最も重要な 3 ～ 5 個の OKR を選択して、最終的なチーム OKR を形成します。

1.4. チーム リーダーは、全員のチーム OKR を統合し、承認のために上層部に提出します.必要に応じて、上層部の意見に基づいて対応する追加を行い、再度ディスカッション ミーティングを開催します.合意と理解に達した後、最終的なチーム OKR を形成するための最終ドラフトが作成されます。

2. 個人の OKR の策定

チームの目標と個人の目標を組み合わせて使用​​する場合にのみ、チームのパフォーマンスは最高のレベルに達することができます。個人の OKR を策定するとき、メンバーはチームの OKR をどのように実行するかを考える必要があります。具体的な手順は次のとおりです。

2.1. まず、メンバーはチームの OKR と個人的な興味を組み合わせ、チームの目標に貢献できる場所について考え、独自の個人的な OKR の下書きを作成します。

2.2. 次に、個人の OKR を匿名の公開 OKR システム プラットフォームに入力して、すべてのチーム メンバーが簡単に閲覧およびコメントできるようにします。

2.3. 2 番目に、リーダーや他のチーム メンバーに OKR に関するコメントを求め、抜けや不適切な箇所がないか確認します。

2.4. 最後に、チームの意見で個人の OKR を更新します。

3. OKR を発表

発表は、決定された OKR をすべての従業員に通知するだけでなく、これらの目標が設定された理由、これらの目標を達成するために会社にとって何を意味するか、これらの目標を一緒に達成するために全員が何をする必要があるかを確認するためのものです。

会社の OKR システムで発表され、誰もが役員や CEO の OKR を含む他の人の OKR を見ることができます。

他人の OKR を理解することで、自分の OKR が他の人に比べて野心的か保守的か、目標の方向性が一貫しているか逸脱しているかを明確に評価し、当事者とコミュニケーションをとって議論することができます。プロセスが実行されます。

4. OKR の実行

主要な結果がある場合、タスクは特定の目標に基づいて分解されるべきであり、各主要な結果は一連のタスクを導き出し、異なるメンバーに割り当てられます。

プロジェクトマネージャーとして、企業リソースの動員と全員の組織化と調整を担当する主要な結果（KR）担当者であり、企業の意思決定者との完全にスムーズなコミュニケーションを維持する必要があります。

実行は、OKR を実装する際のサイクル全体の重要なコア部分であり、実行の焦点は定期的な検査と、必要に応じて通常は週または月に 1 回の調整です。

検査でOKRが考慮する要素は、「目標、現在の進捗状況、発生した問題、問題の原因、必要なサポート、次のステップの計画」をカバーする必要があり、OKRシステムを使用して実装を支援でき、追跡に便利ですプロセス全体は後で。

5. コミュニケーションと採点

各メンバーは、各四半期の開始時に、この四半期の独自の OKR を決定する必要があり、四半期の終了後に、この四半期の作業完了に応じて OKR を採点する必要があります。OKR 採点にはさまざまな形式があります: 0 ~1、1~5、1~10、1~100;

OKR は、挑戦的な目標を設定し、コンフォート ゾーンから抜け出すことをメンバーに奨励しているため、目標を 100% 達成することが必ずしも最高であるとは限りません。最高のスコアは 0.6 ~ 0.7 ポイントです。次のサイクルは調整する必要があり、低いスコアが必ずしも批判されるわけではありませんが、ジョブ分析を通じて、次のサイクルで OKR を改善する方法を見つける必要があります。

6. メンバーのやる気を引き出す

KPI と比較すると、OKR は評価ツールではなく、より有益なツールです. その主な目的は、特定のチームやメンバーを評価することではなく、現在のタスクが何であるかを常に全員に思い出させることです。

7. 前後にループ

このサイクルでは、実行プロセス中に主要な結果を改善する必要があるかどうかを確認できます。ここで、調整できるのは主要な結果であり、目標ではないことに注意してください。

目標を調整することはできません。手段と方法は継続的に改善できます。重要な結果の設定と調整も、マネージャーとメンバー間の直接かつ完全なコミュニケーションの後、コンセンサスでなければなりません。

ANON OKRs-E ナレッジ ベースの詳細

1. 効果的な (ANON OKR-E) 目的の主要な結果と実行の開発
1. 効果的な O: 最初に四半期ごとの目標を策定できます。この目標は、ビジネス開発を促進する目標でなければなりません。この目標は、ビジネス戦略の変革です。通常、数字の代わりに言葉を使用します。この目標の経営陣このように設定された目標を達成できるように、制御可能でなければなりません。

2. 有効な KR を設定する: 一般的な KR は 3 つのタイプに分類されます: 正、有効、臨界値、可能な限り正の KR を設定することをお勧めします。
2. (KR) 主要な結果スコアリング システムを確立する
企業の採点システムが整備されるまで、KR は「明確に定義された」と見なされるべきではありません。

1 = 信じられないほどの結果を得るには、私たちの考え方を変え、別の考え方をする必要があります。
0.7 = 少し難しいですが、不可能ではありません (オッズ 50 から 50)。
0.3 = 「通常の仕事」よりも労力をかけずにできることを知っている;
0 = 進行なし。
3. 進捗状況の追跡/継続的なフィードバック/タイムリーな調整
誰もがチームの目標を理解する必要があり、他のチームの目標の進捗状況も理解する必要があります。これにより、チーム間の作業が容易になり、コラボレーションの効率が向上します。

同時に、メンバーとマネージャー間のリアルタイムのやり取りと、チーム間の価値共有を実現するために、お互いにフィードバックを与える必要があります。
4. 定期的な業績評価
半年ごとまたは年 1 回のパフォーマンス評価 (メンバーの自己評価/メンバーのフィードバック/パフォーマンス インタビュー/パフォーマンスの調整)。パフォーマンス評価の頻度は、四半期ごとまたは毎月にすることはお勧めしません。

OKR の最適な頻度は四半期に 1 回であり、OKR は四半期の終わりにレビューされるため、パフォーマンス評価の頻度が OKR の頻度よりも高い場合、要約と評価の効果がありません。 ある程度重複することもあり、メンバーの負担も大きくなり、メンバーは自分の仕事に集中できなくなります。
